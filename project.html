Why are remote repositories necessary?

Repositories is a place, room or container where something is deposited or stored.

Here is a a simplified breakdown of why Github repositories are important:

  • Central Hub: They hold all project files, documentation and version history
  • Collaboration: Multiple people can work on the project at the same time
  • Track changes: You can see who made what changes and when.
  • Pull Request: Propose updates and get them reviewed before merging.
  • Branches: Work on different features and versions without messing up the main code.

What are the steps involved in creating a project on GitHub?

Details of all the steps:

  • Sign in/up to create an account
  • Create a new repository: clicking the + in the upper right corner
  • Name your repo, add a description and decide if it shoyld be public or private
  • Intialize the repo with a README file you like.
  • Create a .git folder on your local machine
  • Connect your local repository (git) to the remote repo (github)
  • Add files and folders to the Staging Area
  • Commit those files and folders with a message to save them locally
  • Rename your master branch to main branch
  • Push the code from your local repository to the remote repo

Details the relevant git commands relevant to creating a project locally

Here's a cheat sheet for setting up a project locally using Git:

  • Intialize a repository:
  • Setting a new git repository = git init
  • Add files = git add (filename or git add . (dot adds all the files))
  • Commit changes = git commit -m 'Initial commit' (commits added files with a message)
  • Create a Remote repository on GitHu (via GitHub website)
  • Create a new repository and get the repository URL
  • Link local repo to github = git remote add origin (repository-url)
  • Connects your local repo to the GitHub repo
  • Push changes to GitHub = git push -u origin master
  • Pushes your commited changes to the master branch on GitHub. Replace master w. main if necessary
  • Check repo status: git status
  • Shows the current state of your working directory and staging area
  • See commit history: git log
  • Displays the commit history for the repository

Detail how other people can contribute to the project

The contribution begin with the HTML code writing and styling the website

TO contribute to a project in GitHub, other people can follow these steps

  1. The Repository: They can create their own copy of the project by forking the repository. This allows them to make changes without affecting the original project.
  2. Clone the Repository: After forking, they can clone their forked repository to their local machine using Git. This allows them to work on the project offline.
  3. Create a Branch: It's a good practice to create a new branch for their changes. This keeps the main branch clean and allows for easier collaboration.
  4. Make Changes: They can then make their desired changes or additions to the project in their branch.
  5. Commit Changes: Once they are satisfied with their changes, they should commit them with a clear and descriptive message explaining what they did.
  6. Push Changes: After committing, they can push their changes back to their forked repository on GitHub
  7. Open a Pull Request: Finally, they can open a pull request from their branch in the forked repository to the original repository. This is where they can discuss their changes with the project maintainers and get feedback.
  8. Engage in Discussion: They should be open to feedback and ready to make further changes if necessary. Collaboration is key!

By following these steps, anyone can contribute to a project on GitHub and help improve it! If you have any more questions or need further assistance, feel free to ask!
