Why are remote repositories necessary?

The remote repository is cloud based and helps you to store your local files remotely. This helps you access files that aren't already in your computer locally. Other collaborators can also access files by cloning the repositories.

How to create a project locally:

  • Create a project and turn it into a git project by typing: git init
  • This will create a hidden directory called .git that enables communication with GitHub.
  • Once the local project is created, go and create repo for it in Github.

How to create a GitHub repository:

  • Sign in/up to create an account
  • Create a new repository by clicking the upper right + button
  • Name your repo, add a description and make it public
  • Once the GitHub repo is created, you need to connect you local code to the repo.

How to link local and remote repositories:

  • Type in: git remote add origin git@github.com:your-account/repo/.git (Connects your local code to the GitHub repo)
  • Then type: git branch -M main (Renames the current branch to 'main')
  • git push -u origin main (This pushes the local code to the remote repo)

How other people can contribute to the project:

  • They contribute by cloning the project into their local machine.
  • Each project has an endpoint URL that holds the files and directories of the project.
  • In the home page of the repo, look for the green button and copy the SSH code in the tab.
  • Type in: git clone (project-URL) to make a local copy of the project.
  • Now the collaborators can start to add code into the project.
